Démo d’injection SQL - Parcours vulnérable

Suivez le chemin Connexion → Catalogue → Commandes pour observer comment chaque étape est exploitable.

Étapes du parcours

  1. Connexion : usurpation de compte via injection dans la requête WHERE.
  2. Catalogue : exfiltration et manipulation de données via UNION ou mise à jour arbitraire.
  3. Commandes : création de commandes frauduleuses et pollution des données.
